# MAPS 4 层进化到智能体生态（F/C/P/M 升维版）

## F｜事实清单（可检索）
- 层级：L1 使用者（MAPS）→ L2 自动化 → L3 战略伙伴 → L4 智能体生态
- 角色库：创意官/数据官/质检官/风险官/红军AI
- 工具域：工作流(LLM+自动化)、评测集(任务/维度/阈值)、辩论/仲裁器
- 基线术语：胜率=通过率；稳健性=对抗下性能保持率；协同=多体总效能/单体效能max

## C｜概念模型（为什么）
- 个人效能极限 → 通过“多体协作+对抗”获得稳健与创新
- 路径：任务模块化(MAPS) → 自动化串联 → 策略共创 → 生态内生竞争/制衡
- 关键机制：角色分化（偏置互补）× 对抗评审（红军）× 标准化评测（胜率/稳健性）

## P｜SOP（可复现）
- 起点：选 1 个具体任务（如“短视频脚本10条”），定义评测集KPI
  - KPI：可发布率≥60%，原创度≥0.7，事实错误≤1/10
- 角色最小闭环（4体）
  1) 创意官：生成N=50 方案（多样性优先）
  2) 数据官：按历史数据/对标指标打分并Top-K
  3) 红军AI：针对Top-K逐条攻击（逻辑/市场/用户/合规）
  4) 质检官：合并防御与修订，输出候选集
- 决策与收口
  - 仲裁器：基于KPI与红军攻击残留点复评分并出榜单
  - 产出：Top-5 成品 + 攻防记录 + 失败库
- 阈值与分流
  - 若 可发布率<60% → 增强“数据官”训练集或改评分维度
  - 若 红军命中率<50% → 提升攻击模板覆盖/引入外部基准
  - 若 事实错>1/10 → 插入“事实核查体”，强制检索链
- 周期：D0 搭建 → D1 首轮 → D2 复盘与调参 → D7 固化模板

## M｜目标-监控-调整（闭环）
- 周目标：
  - 交付 1 套“4体-攻防-仲裁”流水线，产出≥5 条可发布稿
  - 架构指标：红军命中率≥50%，改稿后错误率↓50%
- 监控（每日 21:30）：
  - 产出数/可发布率/事实错误率/红军命中率/返工时长
- 调整触发器：
  - 命中率<50% → 扩展攻击维度库（再增2类）并升级样例
  - 可发布率<60% → 提升数据官打分权重或换基线语料
  - 返工>30% → 优先优化“创意官多样性→可实现度”约束

## 附：红军AI模板（即用）
- 身份：专业对抗评审，职责=从逻辑/市场/用户/合规四维找致命缺陷
- 约束：每条至少3个攻击点；提供可测试的反证方法；给出致因假设
- 结果：输出“攻击摘要/证据/建议修补/需外部验证点”
