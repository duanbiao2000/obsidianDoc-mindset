Refactoring UI 关于 **font choice（字体选择）** 的实用技巧总结如下：

***

**1. 选用成熟可靠的字体**
- 建议优先选择“系统常用字体”或广泛流行的 Web 安全字体，如 `Inter`, `Roboto`, `Helvetica`, `Arial`, `Georgia`, `SF UI`, `Menlo` 等。
- 这些字体在各平台显示都很稳定，兼容性强。

**2. 简化字体种类**
- 全站通常只用一到两种字体（主字体 + 特殊场景如代码区块字体），过多字体会造成视觉混乱。
- 标题、副标题、正文、按钮均用同一字体即可，通过**粗细、大小、颜色**区分层级。

**3. 合理对比与层级**
- 利用字体的**font-weight（粗细）**和**font-size（大小）**，而不是依赖多种字体，来制造视觉层级。
- 标题可以用加粗，大号字体；正文用普通或稍细字体。

**4. 避免花哨和难阅读的字体**
- 尽量不要用手写体、艺术体、大量连笔或复杂造型的字体，保持页面专业和易读。
- 对于正式产品和内容类页面，优先选择易读性强的无衬线体。

**5. 注意字体兼容和性能**
- 使用自定义字体时，确保字体文件体积适中，考虑网络加载速度，避免拖慢页面渲染。
- 保证回退字体配置，避免目标字体无法加载时出现布局错乱。

***

**Refactoring UI 核心建议摘录：**
- “People will never compliment your font choice in a UI, but they’ll definitely notice if it looks wrong.”
- “Just pick a good sans-serif for most content, and a good monospace for code — nothing more, nothing fancy.”

***

**经典流程举例：**
1. **先选基础字体**：比如 `Inter` 或 `Roboto`。
2. **用变量管理字体层级**：
    - 标题用 font-size: 20px, font-weight: 700
    - 正文用 font-size: 16px, font-weight: 400
    - 按钮用 font-size: 16px, font-weight: 600
3. 特殊场合（如代码区块）选用 `Menlo` 或 `Fira Mono`，但仅限技术类产品。

***

**一句话总结：**
**“字体选择要保守，以功能和清晰为导向。宁选大众型，少选个性型，多用层级而非多字体。”**
